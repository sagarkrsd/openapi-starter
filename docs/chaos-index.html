Harness CE API Documentation

Harness Chaos Engineering empowers teams to deliver more resilient systems, helping organizations avoid downtime that can be costly, both financially and in lost reputation. It enables DevOps and SRE teams to purposely create failure scenarios (“chaos”) to identify potential resiliency and reliability issues in their deployments. These scenarios go beyond traditional unit, integration, and system tests, more closely representing what random failures in a production operating environment would look like.

Let's Get Started

Let's Get Started with a sample Chaos Experiment execution

NOTE: These steps assumes you already have an account in Harness.

Please follow the below guidelines to run a sample Chaos experiment such as Nginx Pod Delete:

Sequence in which the APIs should be executed to run a sample chaos experiment such as Nginx Pod Delete (all the APIs to do so are present in folder "Run Pod Delete Chaos Experiment" under Harness CE Postman Collection) -

Before executing these experiments, please gather all the required inputs/variables so that it can be utilized while executing requests -

  1. url - Harness Chaos Engineering server URL based on the Chaos environment against which you are executing these requests i.e., https://app.harness.io/gateway/chaos/manager/api/query, etc.

    • This will be available in API doc introduction section.

  2. account_id - Your Harness account id. Account ID can be retrieved in the following manner:

    • Go to Account Settings in Harness
    • Click on Overview
    • Copy the value of Account Id

  3. org_id - Your Harness Organisation id. Organisation ID can be retrieved in the following manner:

    • Go to Projects in Harness, select the project against which you want to execute these experiments or create a new one.
    • Click on Overview
    • Copy the value of Organization

  4. project_id - Your Harness Project id. Project ID can be retrieved in the following manner:

    • Go to Projects in Harness, select the project against which you want to execute these experiments or create a new one.
    • Click on Overview
    • Copy the value of Id

  5. API-KEY-TOKEN - Your api-key-token generated through Harness platform. API keys and corresponding tokens can be generated in the following manner:

    • Click on MY PROFILE in Harness
    • Go to My API Keys section and click on + API Key button to create a new API Key
    • Enter name for the API Key and click on Save
    • Under the newly created API Key, click on + Token button to generate a new token for this API Key
    • Enter name and select the validity for the token and click on Generate Token
    • Copy the value of the token generated, this can be used as API-KEY-TOKEN value.

APIs to be executed:

  1. RegisterInfra - Register your Infra where you want to run chaos experiments i.e., your kubernetes cluster, etc.

    • Retrieve the infra manifest by forming the URL like url/file/token.yaml
    • Apply the manifest/.yaml file on your infra such as k8s cluster, some machine, etc.
    • Required Inputs to be modified - No input needs to be modified. Note down the infra ID received in response.

  2. ListPredefinedWorkflow (Optional - Required if and only if you want to run a different chaos experiment other than nginx-pod-delete, by default, nginx pod delete experiment manifest is present in CreateChaosWorkFlow API call) - This will list down all the available predefined workflows/experiments that can be run on your infra, please select one of them and copy the manifest from there and make necessary changes such as namespace, etc.

    • Required Inputs to be modified - No input needs to be modified in the Harness CE Postman collection.

  3. CreateChaosWorkFlow - Create a new workflow using the workflow manifest retrieved in earlier step as an input for this API call.NOTE: Manifest for running experiment "nginx-pod-delete" is available by default in this API here.In case you want to run the default chaos experiment present here, prior to running CreateChaosWorkFlow, create a Nginx app in default namespace with label "app=nginx".

    • Required Inputs to be modified - infraID - Provide the infraID retrieved in response of RegisterInfra API call. Note down the workflow ID received in response.

  4. RunChaosExperiment - Once a workflow is created, you can now execute the experiment by providing the workflow id from step3.Required Inputs to be modified - workflowID - provide the id of workflow created and retrieved in response of CreateChaosWorkFlow API call. Once these steps are executed, user can go and check their chaos experiments list, newly launched experiments should be listed there.

Let's go to Harness CE Postman collection to see this in action.
